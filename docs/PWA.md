# PWA: роли, экраны и офлайн-режим

## Что это

`web/` — мобильное PWA-приложение (Vue 3 + Vite + TypeScript) для:

- водителя: работа с активным рейсом и офлайн-статусами точек;
- административных ролей: управление пользователями и рейсами.

## Роли

Роли хранятся в БД как `role_code`:

- `driver` — Водитель
- `logistic` — Логист
- `accountant` — Бухгалтер
- `admin` — Администратор
- `superadmin` — Супер-админ

## Экраны

### 1) Login

- Логин/пароль, получение JWT через `/auth/login`.

### 2) Мой рейс (driver)

- Получение активного рейса: `GET /v1/mobile/routes/active`.
- Принятие рейса: `POST /v1/mobile/routes/{id}/accept`.
- Смена статуса точки локально (оптимистично) и запись в outbox.

### 3) Пользователи (admin/superadmin)

- Таблица пользователей: login, ФИО, телефон, роль, active.
- Создание пользователя.
- Редактирование login/role/password/full_name/phone.
- Блокировка/разблокировка через `is_active`.

### 4) Рейсы (admin/superadmin/logistic/accountant)

- Список рейсов.
- Создание рейса с назначением водителя и точками.
- Просмотр деталей рейса.
- Переназначение водителя.

## Офлайн-логика (водитель)

Dexie/IndexedDB хранит:

- snapshot активного рейса;
- outbox событий по точкам.

При потере сети:

1. смена статуса сохраняется локально;
2. события накапливаются в outbox.

При восстановлении сети:

1. отправка batch в `POST /v1/mobile/events:batch`;
2. удаление применённых/дубликатов;
3. обновление snapshot с сервера.

Идемпотентность на сервере: `(user_id, device_id, client_event_id)`.

## Установка PWA

После запуска `web` сервиса:

1. откройте PWA в мобильном браузере;
2. добавьте на домашний экран (Add to Home Screen);
3. приложение будет работать как standalone.

Иконки манифеста:

- `web/public/icons/icon-192.png`
- `web/public/icons/icon-512.png`
- `web/public/logo.jpg` (дополнительная иконка).
