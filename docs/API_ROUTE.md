# Добавление рейса по API

API позволяет создавать рейсы в боте DMK по HTTP (аналогично добавлению расчёта ЗП). Не все поля обязательны.

## Аутентификация

Все запросы к API требуют заголовок с ключом:

```
X-API-Key: <ваш_ключ>
```

Ключ задаётся в переменной окружения `API_KEY` на сервере (файл `.env`).

---

## Endpoint: создание рейса

**POST** `/api/route`

**Content-Type:** `application/json`

### Обязательные поля

| Поле           | Тип    | Описание |
|----------------|--------|----------|
| `route_id`     | string | Уникальный номер рейса (например, `00ДМ-000524` или произвольный идентификатор). Не должен совпадать с уже существующим рейсом. |
| Водитель       | —      | Нужно указать **одно** из полей: `driver_name` или `driver_tg_id`. |
| `driver_name`  | string | ФИО водителя **полностью**, как записан в системе (точное совпадение с полем в БД). |
| `driver_tg_id` | number | Telegram ID водителя (число). |

### Необязательные поля рейса

| Поле                  | Тип    | Описание |
|-----------------------|--------|----------|
| `number_auto`         | string | Номер ТС. |
| `temperature`         | string | Температурный режим. |
| `dispatcher_contacts` | string | Контакты диспетчера. |
| `registration_number` | string | Номер для регистрации. |
| `trailer_number`      | string | Номер прицепа. |

### Точки маршрута (необязательно)

| Поле       | Тип    | Описание |
|------------|--------|----------|
| `points`   | array  | Список точек. Если не передан или пустой — рейс создаётся без точек (можно добавить позже в боте). |

Каждый элемент массива `points` — объект:

| Поле         | Тип    | Обязательность | Описание |
|--------------|--------|----------------|----------|
| `type_point` | string | да             | Тип точки: `"loading"` (загрузка) или `"unloading"` (выгрузка). При ином значении сохраняется как загрузка. |
| `place_point`| string | да             | Место (адрес/название точки). |
| `date_point` | string | да             | Дата в формате **дд.мм.гггг** (например, `26.02.2026`). |

---

## Коды ответов

| Код | Описание |
|-----|----------|
| 200 | Рейс успешно создан. |
| 400 | Ошибка в запросе: не указан водитель или передан не водитель. |
| 401 | Неверный или отсутствующий API-ключ. |
| 404 | Водитель с указанным `driver_name` или `driver_tg_id` не найден. |
| 409 | Рейс с таким `route_id` уже существует. |
| 500 | Внутренняя ошибка при создании рейса. |

---

## Пример запроса (минимальный)

Рейс только с номером и водителем (без точек и без доп. полей):

```json
{
  "route_id": "00ДМ-000600",
  "driver_name": "Иванов Иван Иванович"
}
```

Или с указанием водителя по Telegram ID:

```json
{
  "route_id": "00ДМ-000600",
  "driver_tg_id": 123456789
}
```

## Пример запроса (полный)

Рейс со всеми полями и двумя точками (загрузка и выгрузка):

```json
{
  "route_id": "00ДМ-000601",
  "driver_name": "Иванов Иван Иванович",
  "number_auto": "А123ВС456",
  "trailer_number": "ОО 123",
  "temperature": "+2...+6",
  "dispatcher_contacts": "+7 999 123-45-67",
  "registration_number": "РЕГ-001",
  "points": [
    {
      "type_point": "loading",
      "place_point": "Склад ул. Ленина, 1",
      "date_point": "10.03.2026"
    },
    {
      "type_point": "unloading",
      "place_point": "Магазин ул. Мира, 10",
      "date_point": "11.03.2026"
    }
  ]
}
```

## Пример вызова (curl)

```bash
curl -X POST "http://localhost:8010/api/route" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: xhQWxxWGRYFlOrSMbC7hzo8YBfUXwudtRextb3id3GiBe5IvFMfpOet58P4saLAV" \
  -d '{
    "route_id": "00ДМ-000602",
    "driver_name": "Иванов Иван Иванович",
    "number_auto": "А123ВС456",
    "points": [
      {
        "type_point": "loading",
        "place_point": "Склад 1",
        "date_point": "15.03.2026 14:00"
      }
    ]
  }'
```

Успешный ответ (200):

```json
{
  "id": "00ДМ-000602",
  "message": "Route created successfully",
  "points_count": 1
}
```

## Важно

1. **Водитель** должен быть заранее заведён в системе и иметь роль «Водитель». Поиск по ФИО идёт по **полному совпадению** имени в БД.
2. **`route_id`** должен быть уникальным в рамках всех рейсов.
3. После создания рейса водителю **отправляется уведомление в Telegram** (как при создании рейса вручную в боте): текст «У вас новый рейс» с полными данными рейса и точками; при отсутствии активного рейса у водителя — кнопка «✅ Принять рейс».
4. Рейс создаётся со статусом `new` (не принят водителем). Точки создаются со статусом `new`.
